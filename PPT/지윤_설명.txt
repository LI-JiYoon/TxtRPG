처음에 ref를 안쓰고 주소참소 형식으로 복사본을 넘겨 돌렸더니 몬스터를 다 잡았는데도 게임이 정상적으로 종료가 안되더라
나는 이 이유를 복사본을 넘겨줘서 몬스터 처치 판별에 문제가 생긴건가 해서 ref 를 붙이고 넘겨줬더니 잘 돌아가더라. 그래서 그냥 이렇게 썼다

clone 사용 이유

clone 없이 몬스터 대기열 배열을 만들 경우, 중복해서 뽑인 동일한 종류의 몬스터가 있다고 할 때, 분명 하나의 대상에만 데미지를 줬는데 같은 종류의 다른 대상도 같이 데미지가 다는 버그가 있었다.


c# Clone() 메서드를 사용하지 않으면 기본적으로 객체는 참조로 전달된다. 이는 객체의 주소를 복사한다는 뜻이다.
즉,  두 변수가 같은 객체를 가리키게 되고, 한 변수에서 객체의 값을 변경하면 다른 변수에서도 그 변경이 반영되기 때문에 아까와 같은 오류가 발생하는 것이다.

clone은 C#에서  객체의 내용을 동일하게 복사해서 새로운 객체를 만드는 기능이다. 
이를 통해 원본 객체와는 독립적인 별도의 객체를 만들 수 있기에 clone 메서드를 사용하여 몬스터 대기열에 들어갈 몬스터 객체들을 생성했다,
